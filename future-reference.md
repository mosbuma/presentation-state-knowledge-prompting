# Presentation bootstrap — structure reference

Author the deck in **`presentation-outline.md`** (slide titles, bullets, frontmatter). Use any outline shape you prefer — acts, themes, or a simple linear list.

## Optional supporting files

| File | Role |
| --- | --- |
| **`topics.md`** *(create if needed)* | Checklist, per-slide notes, or long-form script keyed to section / slide titles. |
| **`story.md`** | Spoken narrative; referenced from the outline frontmatter for workflow. |
| **`presentation-outline.md`** | Slide source and input to **`npm run present:slides-com`** (Slides.com export). |

## Optional coherence check

When adding examples, tie them to the same **through-line** (shared questions or themes you define). You do not need every slide to restate the frame.

## Git

If this matches your workflow: avoid Cursor-style commit trailers on check-in.
